Database Schema & Entity Relationship Diagram (ERD)

This document outlines the complete database schema for the School Finance Administration System (SFAS).

1. Entity Relationship Diagram (ERD)

erDiagram USER ||--o{ REQUEST_FUND : "submits" USER ||--o{ REPORT : "generates" USER ||--o{ VENDOR : "registers" USER ||--o{ COLLECTION : "enters" USER ||--o{ BILL : "reviews" USER ||--o{ REQUEST_FUND : "reviews" VENDOR ||--o{ BILL : "provides" USER { uuid id PK string name string email UK string password enum role "ADMIN, FINANCE_OFFICER" timestamp created_date } VENDOR { uuid id PK string name string contact_email string phone_number string tax_id string address uuid created_by FK timestamp created_date } BILL { uuid id PK uuid vendor_id FK uuid created_by FK string title decimal amount decimal tax datetime date string image_path string description enum status "PENDING, APPROVED, REJECTED" uuid reviewed_by FK timestamp created_date } REQUEST_FUND { uuid id PK uuid created_by FK string title string description decimal amount_of_fund enum urgency_level "LOW, MEDIUM, HIGH, CRITICAL" datetime date string image_path enum status "PENDING, APPROVED, REJECTED" uuid reviewed_by FK timestamp created_date } COLLECTION { uuid id PK date date enum type "DAILY, WEEKLY, MONTHLY, ANNUAL" decimal amount uuid created_by FK timestamp created_date } REPORT { uuid id PK uuid created_by FK datetime date string title string description enum type_of_report "EMAIL, IN_APP" timestamp created_date }

2. Detailed Database Schema (Tables)

users

ColumnTypeConstraintsDescription
idUUIDPRIMARY KEYUnique identifier for the user.
nameVARCHARNOT NULLFull name of the user.
emailVARCHARNOT NULL, UNIQUEUsed for login.
passwordVARCHARNOT NULLBCrypt hashed password.
roleVARCHARNOT NULLEnum: ADMIN or FINANCE_OFFICER.

vendors

ColumnTypeConstraintsDescription
idUUIDPRIMARY KEYUnique identifier for the vendor.
nameVARCHARNOT NULLName of the vendor/company.
contact_emailVARCHARNOT NULLContact email.
phone_numberVARCHARNOT NULLPhone number.
tax_idVARCHAR-Tax Identification Number.
addressVARCHAR-Physical address.
created_byUUIDFOREIGN KEYLinks to users.id.

bills

ColumnTypeConstraintsDescription
idUUIDPRIMARY KEYUnique identifier.
titleVARCHARNOT NULLShort title of the bill.
vendor_idUUIDFOREIGN KEYLinks to vendors.id.
amountNUMERICNOT NULLTotal amount.
taxNUMERIC-Applied tax.
dateTIMESTAMPNOT NULLDue/issuance date.
image_pathVARCHAR-Reference to the bill proof.
statusVARCHARNOT NULLEnum: PENDING, APPROVED, REJECTED.
created_byUUIDFOREIGN KEYLinks to users.id.
reviewed_byUUIDFOREIGN KEYLinks to users.id.

request_funds

ColumnTypeConstraintsDescription
idUUIDPRIMARY KEYUnique identifier.
titleVARCHARNOT NULLTitle of the request.
amount_of_fundNUMERICNOT NULLRequested budget.
urgency_levelVARCHARNOT NULLEnum: LOW, MEDIUM, HIGH, CRITICAL.
dateTIMESTAMPNOT NULLRequired date.
statusVARCHARNOT NULLEnum: PENDING, APPROVED, REJECTED.
created_byUUIDFOREIGN KEYLinks to users.id.
reviewed_byUUIDFOREIGN KEYLinks to users.id.

collections

ColumnTypeConstraintsDescription
idUUIDPRIMARY KEYUnique identifier.
dateDATENOT NULLCollection date.
typeVARCHARNOT NULLEnum: DAILY, WEEKLY, MONTHLY, ANNUAL.
amountNUMERICNOT NULLAmount collected.
created_byUUIDFOREIGN KEYLinks to users.id.

reports

ColumnTypeConstraintsDescription
idUUIDPRIMARY KEYUnique identifier.
dateTIMESTAMPNOT NULLGeneration date.
titleVARCHARNOT NULLReport title.
type_of_reportVARCHARNOT NULLEnum: EMAIL, IN_APP.
created_byUUIDFOREIGN KEYLinks to users.id.
